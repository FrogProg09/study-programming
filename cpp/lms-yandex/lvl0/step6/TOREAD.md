# Char, или Почему символы — это цифры в маскарадных костюмах

На прошлых уроках мы познакомились с разными числами и даже научились переводить их из целых в дробные. Все числа на свете, к сожалению, узнать не выйдет — их бесконечное количество, в отличие от символов. Все они объединяются в алфавиты, причём не только привычный нам кириллический (А, б, в, ...) или латинский (A, b, c, ...), но и алфавиты специальных символов или эмодзи.

*У спецсимвола @, пожалуй, самая сложная судьба: нам привычно называть его «собакой», в Израиле он носит имя «штрудель», в Сербии — «чокнутая А», в Дании — «А с хоботом», а в Германии — «обезьянка».*

Есть множество таких алфавитов — таблиц кодировок, где у каждого символа есть свой номер. Самая древняя из них — ASCII, созданная в 1963 году для телетайпов. Её авторы даже не догадывались, что однажды люди захотят ставить 😂 вместо того, чтобы печатать LOL (да и о самой аббревиатуре в те времена никто не слышал). Но именно с ASCII началась эра цифровых символов

Этот алфавит состоит из 128 символов:
    - 33 управляющих символа (0-31, 127) - невидимые симовлы (например *\n* для новой строки)
    - 94 печатных символа (32-126) - буквы (английского алфавита), цифры и знаки. Например, код символа '0' - 48.
    
*Небольшой лайфхак для изменения регистра английского текста: код ’A’ = 65, ’a’ = 97, ’B’ = 66, ’b’ = 98 и т. д. Разница в 32 пункта — это длина латинского алфавита с добавлением 6 специальных символов.*

Потом на основе ASCII начали появляться другие кодировки, которые к существующим символам добавляли новые, с кодами 128–255. Чаще всего эти кодировки содержали буквы локальных алфавитов (например, Windows-1251, ISO-8859-1).

Для хранения информации об ASCII-символах в C++ используется тип данных char, и это тот ещё персонаж (от англ. character). В него можно занести только один символ — и никаких русских букв или эмодзи 😭

``` cpp
#include <iostream>

using namespace std;

int main() {
    char letter = 'A';
    cout << "Символ: " << letter << endl;
    cout << "Его ASCII-код: " << int(letter) << endl;
    cout << "ASCII-код следующего символа: " << int(letter) + 1 << endl;
    cout << "Cледующий символ: " << char(int(letter) + 1) << endl;

    return 0;
}
```

В переменную letter мы записали символ A (первую букву английского алфавита, не русского), затем получили код этого символа, используя приведение типов (как в прошлом уроке). А затем, воспользовавшись тем, что все элементы в ASCII-таблице идут по порядку (то есть за заглавной A строчной s не будет), получили следующий символ.

*К сожалению, в кодировках с языками, отличными от английского, всё не так чётко: например, буква Ё стоит чаще всего раньше русской А, а ё — позже я (остальные по порядку).*

Часто этим лайфхаком пользуются, когда создают шифры. Например, шифр Цезаря (или код Цезаря), древний способ шифрования, который работает по принципу: «Давай сдвинем буквы и запутаем врагов». Да, сейчас он смотрится наивно и расшифровывается элементарно, но чуть более двух тысяч лет назад был хорош!

Принцип шифра таков: каждая буква заменяется другой, которая находится от искомой на фиксированном расстоянии. Например, если сдвигать алфавит вправо на 3, то А превращается в Г, Б в Д и так далее. Простенько, но со вкусом — то есть в стиле самого Гая Юлия!

Но поскольку для типа данных char должно хватать 1 байта памяти, а символы русского языка занимают аж два (настолько он велик и могуч), то при попытке занести букву Ы в переменную char компьютеру резко поплохеет, и компилятор сообщит об ошибке:

```
error: character too large for enclosing character literal type
    4 |     char c = 'Ы';
      |              ^
```

И что же нам делать? Кроме стандартного типа данных char есть ещё один, wchar_t, который может хранить символы из другой, очень популярной таблицы — Юникода. Она-то и содержит всё, что мы любим, — эмодзи и символы разных локальных алфавитов (我们正在学习编程, например) — в одном месте. В отличие от ASCII, который хранит только 128 символов, Unicode может вместить аж 1 111 998. Столько даже пока не придумали, всего занято 155 063 позиций. Так что, возможно, кто-то из нас создаст новый символ и увековечит себя в этой кодировке!

*Символ Ё появился в Unicode только в 1993 году, а до этого его заменяли на Е. Так что любители называть ёжиков «ежиками» не безграмотные, а приверженцы старой школы.*

**wchar_t** — это портал для символов из 1-байтовой (ASCII) в 4-байтовую (Unicode) мультивселенную. Чтобы всё прошло гладко, нужно вместо уже привычных **cin** и **cout** использовать **wcin** и **wcout**. А перед каждой фразой добавлять символ **L**.

Но и это не самое важное! Самое важное — подключить библиотеку <local> и указать, что мы хотим использовать именно локаль (то есть набор языковых стандартов) с поддержкой русских букв и эмодзи.

```cpp
#include <iostream>
#include <locale>

using namespace std;

int main() {
    setlocale(LC_ALL, ""); // Устанавливаем локаль для корректного вывода. Сработает, если у вас русскоязычная ОС, иначе нужно указать явно
    wchar_t emojiHappy= L'😛';
    wchar_t russianLetter = L'Ы';
    wchar_t emojiUnknown = wchar_t((int)emojiHappy +1);
    wcout << L"Вот буква: " << russianLetter << endl;
    wcout << L"Вот следующая буква: " << wchar_t((int)russianLetter +1) << endl;
    wcout << L"Вот эмодзи: " << emojiHappy << endl;
    wcout << L"Вот следующее эмодзи: " << emojiUnknown << endl;

    return 0;
}
```

Вот следующая буква: Ь
Вот эмодзи: 😛
Вот следующее эмодзи: 😜
